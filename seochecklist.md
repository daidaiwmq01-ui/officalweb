Cursor 前端 SEO 质量检查单
请作为一名 SEO 专家，根据以下检查点（基于《SEO实战密码》第5版）审查当前前端项目的代码。请指出不符合 SEO 最佳实践的地方，并给出修改建议。
1. 页面头部标签 (Head Tags) 检查
• Title 标签优化
    ◦ 检查点：检查所有页面是否都有 <title> 标签。
    ◦ 标准：Title 必须唯一，不能重复。核心关键词应放在最前面。长度建议控制在 30 个中文字符（或 60 个字节）以内，避免被搜索引擎截断,。
    ◦ 代码特征：<title>...</title>
• Meta Description 标签
    ◦ 检查点：检查页面是否包含 <meta name="description">。
    ◦ 标准：每个页面必须有独特的描述。长度建议控制在 77 个中文字符（约 150 个英文字符）以内。应包含目标关键词，并形成通顺的句子以吸引点击,。
    ◦ 代码特征：<meta name="description" content="...">
• Canonical 标签 (规范化网址)
    ◦ 检查点：检查是否包含规范化标签，防止重复内容问题。
    ◦ 标准：每个页面应指向其自身的规范化 URL（绝对路径），特别是当 URL 包含动态参数时,。
    ◦ 代码特征：<link rel="canonical" href="...">
• 移动端适配标签
    ◦ 检查点：检查是否包含 Viewport 标签。
    ◦ 标准：响应式设计是 Google 和百度推荐的方式。必须声明 viewport 以适应不同屏幕宽度,。
    ◦ 代码特征：<meta name="viewport" content="width=device-width, initial-scale=1.0">
2. 页面结构与语义化 (Semantic Structure)
• H 标签层级
    ◦ 检查点：检查 <h1> 到 <h6> 的使用情况。
    ◦ 标准：每个页面应只有一个 <h1> 标签（通常包含核心关键词）。H 标签应构建清晰的文档层级结构，不要滥用 H 标签仅为了样式,。
    ◦ 代码特征：<h1>, <h2> 等。
• 图片 Alt 属性
    ◦ 检查点：检查所有 <img> 标签是否都有 alt 属性。
    ◦ 标准：有意义的图片（如产品图、Logo）必须包含 alt 描述文字，并包含关键词。装饰性图片可留空但属性必须存在,。
    ◦ 代码特征：<img src="..." alt="描述内容">
• 链接与导航
    ◦ 检查点：检查导航菜单和重要链接的实现方式。
    ◦ 标准：必须使用 <a href="..."> 的 HTML 文本链接。避免使用 JavaScript (onclick)、Flash 或图片作为主要导航，否则爬虫无法抓取,。
    ◦ 代码特征：检查是否存在非 <a> 标签的跳转逻辑。
3. URL 结构与路径 (URL Structure)
• URL 静态化与规范
    ◦ 检查点：检查路由配置或链接生成逻辑。
    ◦ 标准：URL 应尽量静态化（伪静态），避免过多参数（如 ?id=123）。单词间应使用连词符 - 而非下划线 _。字母应全部小写,,。
    ◦ 代码特征：检查路由文件或 Link 组件的路径生成。
• 绝对路径 vs 相对路径
    ◦ 检查点：检查内部链接的路径格式。
    ◦ 标准：虽然两者皆可，但绝对路径有助于预防被采集时的链接丢失和网址规范化问题,。
    ◦ 代码特征：href="https://domain.com/..." vs href="../..."
4. 抓取与索引控制 (Crawling & Indexing)
• Nofollow 的使用
    ◦ 检查点：检查出站链接或非重要页面链接（如“登录”、“注册”）。
    ◦ 标准：对于广告链接、用户生成内容（UGC）或不想传递权重的非必要页面，应添加 rel="nofollow",。
    ◦ 代码特征：rel="nofollow"
• Robots.txt 与 Sitemap
    ◦ 检查点：检查项目根目录是否存在 public/robots.txt 和 sitemap.xml 生成逻辑。
    ◦ 标准：确保没有错误地禁止搜索引擎抓取重要 CSS/JS 文件。Sitemap 应包含所有重要页面的 URL,。
5. 性能与用户体验 (Core Web Vitals)
• 加载速度与代码精简
    ◦ 检查点：检查是否存在未使用的 CSS/JS 或过大的资源包。
    ◦ 标准：精简代码，将 CSS/JS 放在外部文件中。页面速度是重要排名因素（特别是 LCP 指标）,。
    ◦ 建议：检查是否使用了图片懒加载（Lazy Loading）和下一代图片格式（WebP）。
6. 特殊页面检查
• 404 页面
    ◦ 检查点：检查是否配置了自定义 404 错误页。
    ◦ 标准：当访问不存在的路由时，服务器或前端路由应正确返回 404 状态（或显示 404 视图），而不是重定向到首页（软 404）,。
